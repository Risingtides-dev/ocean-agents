# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in this repository, please report it
through **GitHub Private Vulnerability Reporting**:

1. Go to the repository's **Security** tab.
2. Click **Report a vulnerability**.
3. Provide a clear description, reproduction steps, and impact assessment.

Do not open a public issue for security vulnerabilities.

We acknowledge reports within a reasonable timeframe and coordinate disclosure
timing with you.

## Scope

This policy covers the source code and released artifacts in this repository.
Vulnerabilities in dependencies should be reported to their upstream maintainers.

## Safe harbor

Good-faith security research that follows this policy is welcomed. We will not
pursue legal action against researchers who respectfully report genuine issues.
