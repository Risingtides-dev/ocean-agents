# Support

## Getting help

- **Bugs and feature requests** — open a [GitHub Issue](../../issues).
- **Security vulnerabilities** — see [SECURITY.md](SECURITY.md). Do not use
  public issues for security reports.
- **Code questions** — review the nearest `AGENTS.md` for ownership and
  contracts, then open an issue if something is unclear.

## Before opening an issue

1. Search existing issues to avoid duplicates.
2. Check the repository's `AGENTS.md` and `README.md` for documented behavior.
3. Include the relevant version, platform, and reproduction steps.
